# Markdown文件转心理学报格式word文档

## 这是什么

把一个 `.md` 格式的心理学论文草稿，一键生成符合《心理学报》排版规范的 `.docx` 文件。

格式规格参照期刊官网投稿要求和发布的文章的实际视觉效果。

## 文件说明

```
即插即用包/
├── gen_docx_dualcol.py          ← 核心脚本
├── AI撰写md提示词.txt            ← 格式化 AI 的详细提示词
├── 模板/
│   ├── 心理学报模板1.1.dotx      ← 原始模板（供 Word 手动排版参考）
│   └── 心理学报模板1.1.dotx      ← 原始模板（开发参考）
├── 示例/
│   └── 示例论文_尽责性与工作记忆.md ← 格式范例（展示所有标点正确写法）
└── README.md                    ← 本文件
```

## 环境要求

- Python 3.8+
- python-docx：`pip install python-docx`
- 推荐安装 pypinyin（用于中文参考文献拼音排序）：`pip install pypinyin`

未安装 pypinyin 时，中文文献仍会排在英文文献之前，但内部顺序按 Unicode 而非拼音排列。

## 使用方法

### Agent 自动生成

将本包完整丢给 Agent（Cursor / Trae Work / Claude Code 等），Agent 会：

1. 阅读 `AI撰写md提示词.txt` 理解完整格式规范
2. 阅读 `示例/示例论文_尽责性与工作记忆.md` 理解 Markdown 结构
3. 生成一篇同格式的 `.md` 论文
4. 执行 `python gen_docx_dualcol.py <论文名>.md` 输出 `.docx`

Agent 无需手动操作 Word，仅需 Python 环境。

### 手动使用

1. 参照 `示例/示例论文_尽责性与工作记忆.md` 写一篇论文
2. 运行：`python gen_docx_dualcol.py 我的论文.md`
3. 在 Word 中完成收尾（见下文）

## Markdown 格式要求

```markdown
# 论文题目

## 摘要

摘要正文……

关键词：关键词1；关键词2；关键词3

## 1 引言

正文段落……

### 1.1 小节标题

正文段落……

## 参考文献

作者. (年份). 文章标题. *期刊名, 卷*(期), 页码. DOI
```

**关键规则：**

| 规则 | 说明 |
|------|------|
| `##` = 一级标题 | `## 1 引言`、`## 2 文献综述` |
| `###` = 二级标题 | `### 2.1 概念定义` |
| 期刊名+卷号一体 `*` 包裹 | `*Child Development, 81*(6)` 而非 `*Child Development*, 81` |
| 书名 `*` 包裹 | `*教育心理学*` |
| 统计符号 `^` 包裹 | `^r^ = 0.28, ^p^ < 0.01`，脚本自动转斜体 |
| 引号用 `「」` 占位 | 脚本自动转为英文弯引号 + Times New Roman |
| 中文标点正常写 | 逗号/分号/冒号/括号脚本自动转英文半角 + TNR |
| 不编造参考文献 | 缺失信息宁可留空 |
| 正文不要有 `##` `###` | 会被误解析为标题 |

详细格式规范见 `AI撰写md提示词.txt`。

## 在 Word 中收尾

生成后需要检查：

1. **替换个人信息**：`[你的姓名]` `[你的学校]` 等 → 真实信息（Ctrl+H）
2. **检查斜体**：统计符号（`^r^` `^p^` `^t^` `^F^` `^M^` `^SD^` 等）和参考文献期刊名+卷号是否均已斜体
3. **搜索 `^`**：确保无残留标记（正常已剥离，残留说明标记写法有误）
4. **搜索 `##` `###`**：确保正文中无意外井号标题
5. **三线表**：如有表格，手动应用三线表样式

## 故障排查

### 常见问题与归因

很多问题不是程序 bug，而是 AI 输出的 .md 文件格式不对。诊断方法：

| 现象 | 大概率原因 | 归因 |
|------|-----------|------|
| 正文中出现孤立的加粗大号标题 | .md 中 `##` 或 `###` 误入正文段落 | **AI 格式错误** → 修正 .md |
| 期刊名斜体但卷号正体 | .md 写了 `*Child Development*, 81` 而非 `*Child Development, 81*` | **AI 格式错误** → 修正 .md |
| 统计符号没斜体 | .md 没加 `^` 标记，直接写了 `r = 0.28` | **AI 格式错误** → 修正 .md |
| Word 搜到残留的 `^` 符号 | `^` 不成对（少写或多写了） | **AI 格式错误** → 修正 .md |
| 引号是直引号 `""` 而非弯引号 | AI 没用 `「」` 占位符 | **AI 格式错误** → 修正 .md |
| 标点仍是全角中文标点 | 脚本只替换逗号/分号/冒号/括号，罕见标点未覆盖 | 大概率 **AI 格式错误** |
| 参考文献排序错乱 | 缺 pypinyin 或条目格式畸形 | **程序问题** 或 AI 格式错误 |
| 编码报错 / 乱码 | .md 不是 UTF-8 | **用户/AI 错误** |
| 程序直接崩溃 | .md 解析异常（分隔符畸形等） | 把报错信息发给 AI 排查 |

如果 `gen_docx_dualcol.py` 运行报错：
- 检查 `.md` 文件是否是 UTF-8 编码
- 检查 `.md` 中是否有畸形的 `##` / `###`（正文中 `P#5` 这种会被误解析）

## 输出格式规格

| 元素 | 字体 | 字号 | 行距 | 段后 |
|------|------|:----:|------|:----:|
| 题目 | 黑体 | 22pt | 多倍 1.16 | 8pt |
| 作者 | 仿宋 | 14pt | 多倍 1.16 | 8pt |
| 单位 | 宋体 | 9pt | 多倍 1.16 | 8pt |
| 摘要 | 宋体 | 10.5pt | 多倍 1.16 | 8pt |
| 关键词 | 宋体 | 10.5pt | 多倍 1.16 | 24pt |
| 一级标题 | 宋体 | 14pt | 多倍 1.16 | 8pt |
| 二级标题 | 黑体 | 10.5pt | 多倍 1.16 | 0pt |
| 正文 | 宋体 | 10.5pt | 多倍 1.16 | 0pt |
| 参考文献标题 | 黑体 | 12pt | 多倍 1.16 | 8pt |
| 参考文献条目 | 宋体 | 9pt | 多倍 1.16 | 0pt |

**页面：** A4，上下 2.54cm，左右 2.00cm  
**分栏：** 正文 + 参考文献双栏，间距 435 twips  
**参考文献：** 悬挂缩进 0.63cm，中文在前按拼音，英文在后按字母
